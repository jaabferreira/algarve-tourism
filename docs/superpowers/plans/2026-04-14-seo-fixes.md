# SEO Fixes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix all non-slug SEO issues across both Atlantis Tours and Algarve & You sites — meta descriptions, page titles, H1s, privacy page, category pages, footer links, structured data, and sitemap.

**Architecture:** Both sites are Astro 5 static builds in a monorepo with a shared package. SEO metadata flows through page components → site Layout → shared PageLayout → shared SEO.astro. Translation keys in `packages/shared/src/i18n/` provide localized strings. The i18n system is typed — new keys must be added to the `TranslationStrings` interface and all 4 locale JSON files.

**Tech Stack:** Astro 5, TypeScript, pnpm workspaces, @astrojs/sitemap, JSON-LD structured data

### Task 6: Create Atlantis boats category page

**Files:**
- Create: `packages/atlantis/src/pages/[locale]/tours/[category]/index.astro`

Note: Atlantis `config.fh.itemPks` only allows 10 specific products through, and all 10 are in the `"boats"` category. The other categories (gastronomy, land-tours, transfers, spa) exist in the shared FareHarbor data but are only shown on Algarve & You. So Atlantis keeps `categories: ["boats"]` — only one category page is generated.

- [ ] **Step 1: Create Atlantis category page**

Create `packages/atlantis/src/pages/[locale]/tours/[category]/index.astro`:

```astro
---
import type { Locale } from "@algarve-tourism/shared";
import { LOCALES, t, getLocalePath } from "@algarve-tourism/shared";
import type { TranslationKey } from "@algarve-tourism/shared";
import Layout from "../../../../layouts/Layout.astro";
import Header from "@algarve-tourism/shared/components/Header.astro";
import Footer from "@algarve-tourism/shared/components/Footer.astro";
import ProductGrid from "@algarve-tourism/shared/components/ProductGrid.astro";
import WhatsAppButton from "@algarve-tourism/shared/components/WhatsAppButton.astro";
import { config } from "../../../../config.js";
import { loadItems } from "../../../../data.js";

export function getStaticPaths() {
  return LOCALES.flatMap((locale) =>
    config.fh.categories.map((category) => ({
      params: { locale, category },
    })),
  );
}

const locale = Astro.params.locale as Locale;
const category = Astro.params.category as string;
const categoryLabel = t(locale, `category.${category}` as TranslationKey);

const items = loadItems(locale).filter((item) => item.category === category);
---

<Layout
  title={categoryLabel}
  description={`${categoryLabel} — ${config.name}`}
  locale={locale}
  path={`/tours/${category}/`}
  config={config}
>
  <Header slot="header" config={config} locale={locale} path={`/${locale}/tours/${category}/`} />

  <section class="section container" style={`padding-top: calc(var(--header-height) + var(--space-12))`}>
    <h1>{categoryLabel}</h1>
    <ProductGrid items={items} brand={config.brand} locale={locale} />
  </section>

  <Footer slot="footer" config={config} locale={locale} />
  <WhatsAppButton phone={config.social.whatsapp} locale={locale} />
</Layout>
```

This is identical to the existing A&Y `[category]/index.astro` — same structure, same imports, same data flow. Only one category page (`/tours/boats/`) is generated because Atlantis config has `categories: ["boats"]`.

  // Generate redirect map for changed slugs
  const redirects: Array<{ locale: string; oldSlug: string; newSlug: string }> = [];

  for (const lang of FH_LANGUAGES) {
    if (lang === "pt") continue;

    const newPath = resolve(dataDir, `${shortname}.${lang}.json`);
    const newItems = JSON.parse(readFileSync(newPath, "utf-8")) as Array<{ slug: string; pk: number }>;
    const newSlugs = new Map(newItems.map((i) => [i.pk, i.slug]));

    for (const [pk, canonical] of canonicalMap) {
      const newSlug = newSlugs.get(pk);
      if (newSlug && newSlug !== canonical.slug) {
        redirects.push({ locale: lang, oldSlug: canonical.slug, newSlug });
      }
    }
  }

  if (redirects.length > 0) {
    const redirectPath = resolve(dataDir, `${shortname}-slug-redirects.json`);
    writeFileSync(redirectPath, JSON.stringify(redirects, null, 2));
    console.log(`Wrote ${redirects.length} slug redirects to ${redirectPath}`);
  }
